자기소개서

자기소개

웹디자인 분야에서 사용자의 복잡한 서비스 흐름을 직관적으로 구조화하는 역량을 바탕으로 다양한 프로젝트 경험을 쌓아왔습니다. 뷰컴즈주식회사 글링개발팀에
서 웹소설 플랫폼과 백오피스, 앱의 UI/UX 디자인을 담당하며 운영·정산·제작 등 분산된 업무 흐름을 기준별로 재구성한 경험이 있습니다. 실무자 인터뷰를 통해 1
1개 기능 중심 메뉴를 5개 업무 흐름으로 재분류하고, 정산 조회와 판매 상세 확인 과정을 필터와 모달 구조로 개선하여 운영자의 정보 탐색 효율을 높였습니다. 반
복 UI 요소를 공통 컴포넌트로 정리해 화면 제작과 유지보수의 효율성도 향상시켰습니다. 팡고지와이주식회사에서는 입시 커뮤니티 앱의 MVP 기획과 UI/UX 디
자인을 3개월 내에 완료하고, QA 이슈 48건을 구조화해 개선 루프를 체계화하였습니다. 실제 사용자 흐름을 고려한 설계로 정보 탐색에서 커뮤니티 참여까지 자
연스럽게 이어지는 경험을 구현하였습니다. 앞으로도 사용자와 비즈니스의 맥락을 깊이 이해하고, 일관된 UI와 명확한 정보 구조를 통해 서비스의 완성도를 높이
는 웹디자이너로 성장하겠습니다.

업무 스타일과 장단점

저는 프로젝트의 목표와 사용자의 실제 이용 맥락을 먼저 파악하고, 이를 IA와 사용자 플로우로 구조화한 뒤 화면 설계로 연결하는 방식으로 일합니다. 기능을 단순
히 나열하기보다, 사용자가 어떤 순서로 정보를 확인하고 어떤 행동으로 이어져야 하는지를 중심으로 화면 구조를 정리하는 편입니다.

gling 프로젝트에서는 독자, 작가, 운영자라는 서로 다른 사용자의 목적을 이해해야 했습니다. 독자에게는 작품 탐색이 비어 보이지 않도록 카드 구조와 기본 표지
시스템을 설계했고, 작가에게는 프로필과 스튜디오를 통해 활동 이력과 정산 흐름을 확인할 수 있도록 구성했습니다. 운영자에게는 복잡한 조회와 정산 업무를 더
효율적으로 처리할 수 있도록 백오피스 메뉴와 데이터 확인 흐름을 재정리했습니다.

저의 강점은 복잡한 정보를 구조화하고, 다양한 이해관계자의 요구사항을 하나의 제품 흐름 안에서 정리하는 능력입니다. 기획, 개발, 운영, 콘텐츠 직군과 협업하
며 요구사항을 조율하고, 디자인 시스템과 화면 정책을 통해 반복되는 커뮤니케이션을 줄이기 위해 노력해 왔습니다.

한편 여러 흐름을 함께 고려하다 보면 초반 설계 범위가 넓어지는 경우가 있었습니다. 이를 보완하기 위해 최근에는 MVP 기준과 우선순위를 먼저 정리하고, 화면
별 체크리스트와 QA 항목을 만들어 핵심 플로우부터 검증하는 방식으로 작업하고 있습니다. PangoGY에서 입시 커뮤니티 앱 MVP를 진행할 때도 QA 이슈 48
건을 구조화해 수정 우선순위를 정리하며 출시 안정화를 지원했습니다. 이 경험을 통해 속도와 완성도의 균형을 맞추는 방법을 계속 개선하고 있습니다.

가장 기억에 남는 프로젝트

가장 기억에 남는 프로젝트는 웹소설 플랫폼 gling의 서비스 구조를 설계한 경험입니다.

gling은 독자가 작품을 탐색하는 플랫폼이면서, 동시에 작가가 자신의 작품을 관리하고 판매·정산 흐름을 확인해야 하는 서비스였습니다. 초기에는 콘텐츠 탐색,
작가 이력, 출판 관리, 정산 확인, 운영자 백오피스가 각각 분산되어 있어 사용자가 전체 흐름을 이해하기 어려운 문제가 있었습니다. 저는 이 문제를 단순한 화면
개선이 아니라 서비스 구조를 다시 정리해야 하는 문제로 보았습니다.

B2C 플랫폼에서는 표지 자산의 완성도와 스타일 편차가 큰 초기 작품군도 탐색 경험이 비어 보이지 않도록 텍스트 중심 카드 구조와 장르별 기본 표지 시스템을
설계했습니다. 또한 작가의 대표작, 활동 이력, 성과 지표를 하나의 프로필 허브로 통합해 독자가 작가의 활동과 신뢰도를 더 직관적으로 확인할 수 있도록 구성했
습니다.

백오피스에서는 운영 실무자 인터뷰를 통해 메뉴 구조와 조회 방식의 병목을 확인했습니다. 기존에는 기능 단위 메뉴가 흩어져 있어 원하는 데이터를 찾기 위해 여
러 메뉴를 오가야 했고, 정산·판매 데이터를 확인하는 과정도 길었습니다. 이를 해결하기 위해 메뉴를 업무 흐름 기준으로 재분류하고, 조건형 조회와 상세 모달 구

조를 적용해 필요한 데이터를 더 빠르게 확인할 수 있도록 개선했습니다.

작가용 모바일 앱에서는 출판 이후 운영과 정산 흐름을 모바일에서 직접 확인할 수 있도록 정산 홈, 출간 상태, 플랜별 출간 경로를 설계했습니다. 이를 통해 작가가
판매 내역과 정산 상태를 빠르게 확인하고, 출간 진행 상황과 다음 작업을 앱 안에서 이어갈 수 있도록 구성했습니다.

이 프로젝트를 통해 저는 UI/UX 디자인이 단순히 화면을 보기 좋게 만드는 일이 아니라, 복잡한 서비스 구조를 사용자 관점에서 다시 해석하고 제품 경험으로 연결
하는 일이라는 점을 배웠습니다. 또한 콘텐츠, 운영, 정산, 개발 관점이 모두 얽힌 프로젝트에서 각 직군의 언어를 이해하고 화면 구조로 정리하는 경험을 쌓을 수
있었습니다.

입사 후 포부

입사 후에는 복잡한 서비스 흐름을 사용자가 이해하기 쉬운 제품 구조로 정리하는 UI/UX 디자이너로 기여하고 싶습니다.

저는 웹소설 플랫폼, 운영·정산 백오피스, 작가용 모바일 앱, 입시 커뮤니티 MVP를 설계하며 다양한 사용자의 목적과 비즈니스 운영 구조를 함께 고려하는 경험을
해왔습니다. 이 과정에서 사용자에게 필요한 정보를 어떤 순서로 보여줄지, 반복되는 업무를 어떻게 줄일지, 다음 행동으로 자연스럽게 이어지는 구조를 어떻게 만
들지 고민해 왔습니다.

입사 후에는 먼저 서비스의 사용자 흐름과 운영 구조를 빠르게 이해하고, IA와 사용자 플로우를 기반으로 문제를 명확히 정의하겠습니다. 이후 기획, 개발, 운영 직
군과 긴밀히 협업하며 화면 정책과 UI 구조를 정리하고, 사용성과 비즈니스 목표가 함께 충족될 수 있는 제품 경험을 만드는 데 집중하겠습니다.

또한 디자인 시스템 구축 경험을 바탕으로 반복되는 UI 요소와 화면 패턴을 공통 기준으로 정리해, 제품의 일관성과 제작 효율을 높이는 데 기여하고 싶습니다. 장
기적으로는 사용자의 문제를 해결하는 것뿐 아니라, 팀이 더 빠르고 명확하게 제품을 개선할 수 있는 디자인 기반을 만드는 디자이너로 성장하고 싶습니다.
