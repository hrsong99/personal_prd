# 구매 후 첫 레슨 예약 플로우 — ASCII 와이어프레임

플로우에 등장하는 모든 화면의 시각 레퍼런스입니다. PRD와 함께 위에서 아래로 읽거나,
문서 하단의 플로우 다이어그램을 훑어보면서 각 요소가 어떻게 연결되는지 파악하세요.

플로우는 **네 개 화면**으로 구성됩니다: 축하 → 예약 유도 → 레벨 + 시간 선택 → 예약 완료.
여기에 세 개의 바텀시트(이탈 리마인더, 레벨 변경, 캘린더)와 두 개의 홈 상태
(예약 안 함, 예약 완료)가 더해집니다.

> **퍼널 진입 조건(먼저 읽어 주세요).** 전체 퍼널은 오직 유저의
> **첫 실제 결제** 시에만 실행됩니다. 체험 레슨, 업그레이드, 갱신, 플랜 변경은
> 해당하지 않습니다. 그 외 결제 케이스는 **화면 1만 노출되고 수동 확인 버튼**이 붙습니다 —
> 예약 유도, 레벨+시간, 보너스 토스트, N1~N5 알림 모두 없습니다.
> 아래 "화면 1 — 변형 B"를 참고하세요.
>
> **7일 철회 예외:** 이전 결제가 (1) 전액 환불되었고, (2) 완료된 레슨이 0회이며,
> (3) 결제일로부터 7일 이내에 환불된 경우, 해당 결제는 퍼널 진입 판정 시
> 존재하지 않았던 것으로 간주합니다. 계정의 모든 이전 결제가 이 3가지 조건을
> 모두 통과하면, 다음 결제는 첫 실제 결제로 간주되어 유저는 전체 퍼널에
> 다시 진입하고 새로운 `purchase_bonus`를 발급받습니다.
>
> 더 복잡한 엣지 케이스(8일 이후 환불, 고객 지원 에스컬레이션 등)에 대해서는
> 여전히 어드민 재지급이 가능하며, 이 경우 퍼널 화면을 다시 재생하지 않고
> 새 마감일 기준으로 홈 토스트 + 알림이 작동합니다.
>
> **현재 판매 중인 플랜 타입:**
> - **무제한 레슨권 (Unlimited)** — 3 / 6 / 12개월에 대해 각각 +21 / +30 / +60일 연장
> - **라이트 루틴 레슨권 (Light Routine, 월8회)** — 3 / 6 / 12개월에 대해 각각
>   +21일 + 5회 / +30일 + 8회 / +60일 + 12회 (기간 연장과 보너스 레슨이 모두 지급됨)

## 화면 1 — 구매 축하

유저가 전체 퍼널 대상인지 여부에 따라 두 가지 변형이 있습니다.

### 변형 A — 첫 실제 구매 (자동 전환, CTA 없음)

```
┌─────────────────────────────────┐

│                                 │
│                                 │
│                                 │
│                                 │
│                                 │
│                                 │
│              ┌─────┐            │
│             ╱       ╲           │
│            │   ✓     │          │ ← 밝은 녹색 원 안에
│             ╲       ╱              어두운 체크 표시
│              └─────┘            │
│                                 │
│       구매가 완료되었어요!       │ ← 굵은 제목
│                                 │
│      영어 무제한 레슨권 12개월   │ ← 회색 서브타이틀
│                                 │    (플랜명, 동적으로 삽입)
│                                 │
│                                 │
│                                 │
│                                 │
```

- 흰색 배경, 버튼 없음, 혜택 카드 없음, 뒤로가기 화살표 없음
- 유저는 상호작용할 수 없음 — 순수하게 잠깐 스쳐 가는 확인 화면
- **화면 2: 예약 유도**로 자동 전환됨

### 변형 B — 그 외 결제 (수동 확인, 홈으로 이동)

재구매, 업그레이드, 갱신, 플랜 변경 등 첫 실제 결제가 아닌 모든 경우에 노출됩니다.
변형 A와 시각적으로 동일하지만, 하단에 단일 CTA가 붙고 자동 전환은 없습니다.

- 자동 전환 없음, 예약 유도 없음, 이탈 리마인더 모달 없음
- `purchase_bonus` 생성 없음, 홈 토스트 없음, N1~N5 없음
- "확인" 탭 시 유저는 곧바로 홈으로 이동

## 화면 2 — 예약 유도 ("메인 화면")

퍼널의 핵심 지점입니다. 유저가 예약 CTA를 탭하도록 밀어붙이는 단일 액션 화면.
레벨이나 시간대는 노출하지 않습니다 — 이는 화면 3에서 다룹니다.

```
┌─────────────────────────────────┐
│  9:27                ▪▪▪ ▿ ▭   │
│   🎉    🎊         🎉    🎊    │ ← 컨페티 애니메이션
│       🎊        🎉              │   계속 떨어지는 효과
│                                 │
│  🎉             🎊      🎉      │
│            ┌────┐               │
│           /  ∞∞ \               │ ← 기분 좋은 포도 마스코트가
│          | (^_^) |              │   포장된 선물 상자를
│           \ 🎁  /               │   들고 있는 모습
│            ╲___╱                │
│   🎊      [legs]      🎊        │
│                                 │
│       첫 레슨 예약해봐요!       │ ← 굵은 제목
│                                 │
│  🎉                       🎊    │
│                                 │
├─────────────────────────────────┤ ← 고정 하단 영역
│  ┌───────────────────────────┐  │
│  │ 🎁    21일 연장 혜택     │  │ ← 혜택 카드
│  │     지금 바로 첫 레슨하면 │  │   (파란색, 아래를
│  │  이용 기간을 연장해 드려요│  │    향하는 말풍선 꼬리)
│  └─────────────▽─────────────┘  │
│  ┌───────────────────────────┐  │
│  │     첫 수업 예약하기      │  │ ← 프라이머리 CTA
│  └───────────────────────────┘  │   (파랑/보라, 항상
│                                 │    활성 상태)
│       혜택 포기하고 나가기      │ ← 약한 회색 텍스트 링크
│              ▔▔▔▔▔              │   → 이탈 리마인더 시트
└─────────────────────────────────┘
```

- **뒤로가기 화살표 없음** — 빠져나갈 수 있는 유일한 경로는 약한 회색 링크뿐
- "첫 수업 예약하기"는 **항상 활성** 상태 (여기서는 시간 선택이 일어나지 않음)
- 탭 시 → **화면 3: 레벨 + 시간 선택**
- "혜택 포기하고 나가기" 탭 시 → **이탈 리마인더 바텀시트**
- 플랜별 혜택 헤드라인:
  - **무제한** (3 / 6 / 12개월): `"21일 연장 혜택"` / `"30일 연장 혜택"` / `"60일 연장 혜택"`
  - **라이트 루틴** (3 / 6 / 12개월): `"21일 연장 + 5회 추가 혜택"` / `"30일 연장 + 8회 추가 혜택"` / `"60일 연장 + 12회 추가 혜택"`
- **그림자** — 혜택 카드와 CTA 버튼 모두 컨페티 배경 위에서 부드러운 드롭 섀도를
  드리워, 공중에 떠 있는 한 쌍처럼 읽히도록 합니다:
  - 혜택 카드: `0 4px 12px rgba(15, 23, 42, 0.08)` — 부드러운 중립 톤
  - CTA 버튼: `0 8px 20px rgba(97, 132, 255, 0.28)` — `#6184FF` 액센트 컬러와
    버튼 배경색에 맞춘 푸른 기가 도는 섀도이며, 카드보다 조금 더 강하게 잡아서
    CTA가 시각적 계층의 최상단에 위치하도록 합니다
  - 두 표면 모두 배경 블러는 적용하지 않습니다

## 이탈 리마인더 바텀시트

오직 예약 유도 화면에서 "혜택 포기하고 나가기"를 탭했을 때만 노출됩니다.

```
┌─────────────────────────────────┐
│   🎉        🎊         🎉       │ ← 어두운 백드롭 뒤로
│                                 │   컨페티와 기분 좋은
│  ░░░░░░░░░░░░░░░░░░░░░░░░░░░░░  │   마스코트가 딤 처리되어
│                                 │   보임
│ ░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░ │
│┌───────────────────────────────┐│
││            ▬▬▬                ││ ← 드래그 핸들
││                               ││
││           ┌────┐              ││
││          /  ∞∞  \             ││ ← 슬픈 포도 마스코트
││         | (T_T)  |            ││   (눈물 맺힌 무한대 눈,
││          \ 🎁  /              ││    떨어진 선물 상자)
││           ╲___╱               ││
││                               ││
││        정말 나가시겠어요?     ││ ← 굵은 제목
││                               ││
││  지금 나가면 이 혜택을        ││ ← 회색 서브타이틀
││  놓칠 수 있어요.             ││   (의도적으로 혜택이
││                               ││    소멸되는 것처럼 표현 —
││                               ││    실제로는 보너스가
││                               ││    계속 유지됨)

││  │     지금 예약하기       │  ││ ← 프라이머리 녹색 버튼
││  └─────────────────────────┘  ││   → 화면 3: 레벨+시간
││                               ││     (앞으로 이동이며,
││                               ││      화면 2로 돌아가는
││                               ││      것이 아님)
││     혜택 포기하고 나가기      ││
││            ▔▔▔▔▔              ││ ← 약한 회색 링크
│└───────────────────────────────┘│   → 홈 (보너스는 여전히 유효)
└─────────────────────────────────┘
```

⚠️ **"혜택을 놓칠 수 있어요" 카피는 리텐션 넛지이지 실제 규칙이 아닙니다.**
유저가 회색 링크를 탭해도 실제로 혜택을 잃지는 않습니다.
마감일은 계속 돌아가고, 홈 토스트도 여전히 노출되며, 필요 시 자동 연장도
그대로 발동되고, 일반 예약 경로로도 보너스를 여전히 받을 수 있습니다.

플랜별 카피 주의 사항:
- 무제한 (Unlimited): `지금 나가면 이용 기간 연장 혜택을 놓칠 수 있어요.`
- 라이트 루틴 (Light Routine): `지금 나가면 이용 기간 연장과 보너스 레슨 혜택을 놓칠 수 있어요.`

**액션 매핑:**
- "지금 예약하기" (녹색) → 화면 3으로 포워드 이동 (유저를 퍼널 밖으로 통과시킴)
- "혜택 포기하고 나가기" (회색 링크) → 홈으로 이탈 (보너스는 여전히 유효)
- 바깥 영역 탭 / 아래로 드래그 → 시트 닫기, 화면 2에 머무름

## 화면 3 — 레벨 + 시간 선택

예약 인터랙션 화면입니다. 화면 2의 "첫 수업 예약하기" 또는 이탈 리마인더
바텀시트의 "지금 예약하기"를 통해 진입합니다.

```
┌─────────────────────────────────┐
│  ←                              │ ← 뒤로가기 → 화면 2
├─────────────────────────────────┤   (예약 유도)
│                                 │
│  ┌────────────┬─────────────┐   │ ← 더블팩 전용:
│  │   영어     │   일본어    │   │   언어 토글
│  └────────────┴─────────────┘   │   (싱글팩에서는 생략)
│                                 │
│  레슨 선택                      │
│  ┌─────────────────────────┐    │
│  │ ▓▓ Start 1          ›   │    │ ← 레벨 카드 (탭해서 변경)
│  │    기초 영어의 첫걸음   │    │
│  └─────────────────────────┘    │
│                                 │
│  ┌─────────────────────────┐    │ ← 체험 레슨 튜터 추천으로
│  │ 체험 레슨 튜터가         │    │   레벨이 세팅된 경우에만
│  │ Start 1 을 추천했어요!   │    │   표시되는 배너
│  │ 다른 레벨도 선택할 수    │    │
│  │ 있어요.                  │    │
│  └─────────────────────────┘    │
│                                 │
│  추천 시간                      │
│  레슨 일정을 선택해 주세요.     │
│  ┌──────────┐ ┌──────────┐      │
│  │ 오늘     │ │ 오늘     │      │ ← 3일 범위 내에서
│  │ 21:00    │ │ 21:30    │      │   가장 가까운 예약
│  └──────────┘ └──────────┘      │   가능 슬롯 6개를
│  ┌──────────┐ ┌──────────┐      │   시간순으로 노출
│  │ 내일     │ │ 내일     │      │
│  │ 10:00    │ │ 11:00    │      │
│  └──────────┘ └──────────┘      │
│  ┌──────────┐ ┌──────────┐      │
│  │4월 21일  │ │4월 21일  │      │ ← 모레 슬롯은 절대
│  │ 06:30    │ │ 07:00    │      │   날짜 레이블 사용
│  └──────────┘ └──────────┘      │
│                                 │
│       [ 다른 시간 보기 ]        │ ← 고스트 버튼
│                                 │   → 캘린더 바텀시트
│                                 │
├─────────────────────────────────┤
│  ┌───────────────────────────┐  │
│  │       예약 확정           │  │ ← 프라이머리 녹색 CTA
│  └───────────────────────────┘  │   시간 슬롯이 선택되기
│                                 │   전까지는 비활성(회색)
└─────────────────────────────────┘
```

- **이 화면에는 혜택 카드가 없습니다** (혜택 카드는 화면 2에 위치)
- **이 화면에는 이탈 링크가 없습니다** ("나가는" 유일한 방법은 뒤로가기 화살표 → 화면 2)
- "예약 확정"은 시간 슬롯이 선택되기 전까지 **비활성** 상태
- 기본 그리드는 현재 선택된 언어 + 레벨 기준으로 **가장 가까운 예약 가능 시간 6개**이며,
  3일 범위 안에서 시간순으로 정렬됩니다
- **슬롯 레이블:** 오늘 → `오늘 HH:MM`, 내일 → `내일 HH:MM`, 모레 → 절대 달력 레이블 (예: `4월 21일 06:30`)
- **그리드 패딩:** 오늘 남아 있는 슬롯이 6개 미만인 경우(심야 구매 + 2시간 예약 컷오프 때문),
  그리드는 내일, 그다음 모레로 이어서 채워지며 6개 슬롯이 다 채워지거나 3일 범위가 소진될 때까지 계속됩니다.
  전체 범위에서 6개 미만이 가능하다면 있는 만큼만 노출되며, 플레이스홀더는 렌더링하지 않습니다
- 언어를 전환하거나 레벨을 변경하면 선택된 날짜/시간이 초기화되고 그리드가 처음부터 다시 계산됩니다
- 시간이 선택된 상태에서 "예약 확정" 탭 시 → **화면 4: 예약 완료**

- 토스트는 GNB의 상단 엣지와 시각적으로 겹쳐서, GNB 아이콘이 블러 영역

**토스트 라이프사이클:**
- 구매 직후 즉시 노출
- 다음 중 하나가 발생할 때까지 세션을 넘나들며 지속됨: 유저 X 버튼 닫기,
  마감일 만료, 혹은 보너스 지급
- 마감일이 자동 연장된 경우 **새 마감일과 함께 다시 노출됨**
  (이전에 닫았더라도)
- **유저가 예약을 현재 혜택 기간 바깥으로 재예약(reschedule)하면 다시 노출됨**
  (이전에 닫았더라도) — 이때 마감 충돌을 확인할 수 있으며, 재예약 스텝에서
  일회성 경고 시트(`이 시간으로 옮기면 혜택을 받을 수 없어요`)가 함께 노출됩니다
- 토스트 본문은 탭이 불가능하며 X 버튼만 상호작용 가능합니다
- **멀티 구매 엣지 케이스** (첫 구매 위에 어드민 재지급이 얹힌 경우):
  동시에 노출되는 토스트는 하나뿐이며, `created_at` 기준 **가장 최근**
  `purchase_bonus`를 따릅니다

**예약 취소 시맨틱:**
- 유저가 예약된 첫 레슨을 재예약 없이 완전히 취소하면, 홈은 **상태 A**로

  `purchase_bonus` 레코드는 건드리지 않으며, 혜택 기간 내에 새로 예약하면
  여전히 보너스를 받을 수 있습니다.

- 버튼은 **"일정 변경"** (고스트) + **"예습하기"** (프라이머리 녹색)
- "입장하기" / 강의장 입장은 이 카드에 **포함되지 않습니다** — 수업 시작
  시간에 가까운 별도의 UI 표면에서 노출되며, 이는 다른 PRD가 담당합니다
- 예약 카드는 **세 개의 라벨 행**(일정 / 레슨 / 튜터)으로 구성되며,
  이는 두 행만 노출하는 예약 완료 화면(화면 4)과 다릅니다

**D-day 뱃지 규칙** (예약 카드 내부, 토스트와는 별개):
- ≥ 24시간 → `D-2`, `D-1`, ...
- 1시간~24시간 → `{N}시간 전`
- < 1시간 → `{N}분 전`

## 혜택 기간 단계

```
purchase                                                   forfeit
   │                                                          │
   │      INITIAL WINDOW              EXTENDED WINDOW         │
   │   (purchase_day + 0→2)        (purchase_day + 3→7)       │
   ├───────────────────────────┬──────────────────────────────┤
   │                           │                              │
 day 0                       day 2                         day 7
                              23:59                         23:59
                                │
                                ↓ 아직 지급되지 않았다면:
                          자동 연장, N5 발송,
                          새 마감일과 함께
                          토스트 재노출 (이전에 닫혔더라도)
```

- 하나의 보너스, 두 단계. 두 단계 모두 동일한 리워드.
- 연장은 day 2가 끝나는 순간 정확히 한 번만, 자동으로 발생합니다.
- day 7이 완료 없이 지나가면 → 보너스는 영구히 소멸됩니다.

## 알림 타임라인

```
day 0          day 1              day 2          day 6              day 7
purchase       morning 9am        end of day     morning 9am        end of day
   │              │                 │              │                   │
   ├──────────────┼─────────────────┼──────────────┼───────────────────┤
   │              │                 │              │                   │
   N1*            N3                N5             N3                  (forfeit)
   on book       (initial)         extension      (extended)
                                     fires
```

- **N1** — 예약 확정 시 발송 (혜택 기간 내 예약이라면 언제든지)
- **N2** — 혜택 기간 밖에서 예약 확정 시 발송 (드문 케이스이며, 일반 예약
  경로 전용)
- **N3** — 총 두 번 발송: day 1 아침과 day 6 아침. 예약 완료 / 보너스 지급
  / 소멸 시에는 발송 차단
- **N4** — 혜택 기간 안에서 첫 레슨이 완료되었을 때 발송 (구매당 1회)
- **N5** — 초기 기간이 소진되지 않은 채 만료되는 순간 1회 발송 (연장 사실을
  알림)

N4가 발송된 이후 → 침묵 (추가 보너스 알림 없음)
day 7 소멸 이후 → 침묵

### 알림 카피 드래프트

아래는 **PRD와 완전히 동일한 드래프트**입니다 — 두 문서가 서로 어긋나지
않도록 와이어프레임 문서도 동일한 카피를 그대로 포함합니다. 모든 알림톡은
포도의 기존 템플릿(`pd_reg_infinity_2` → `예습하러 Go`, `레슨권 만료 D-1` →
`일기장 몰래보기`)에서 이미 사용 중인 패턴에 맞춰 하단에 웹 링크 버튼 하나만
가집니다.

**폴백 룰.** `pd_reg_weeklyclass_2` / `pd_reg_infinity_2`는 활성
`purchase_bonus`가 없는 유저에게는 기존 그대로 유지됩니다. 새 N1/N2는
`PodoScheduleServiceImplV2.book()` 내부에서 유저에게 활성 미지급 상태의
`purchase_bonus`가 있을 때에만 기존 템플릿을 대체합니다.

**사용되는 변수:**
- `{studentName}`, `{subjectName}`, `{Lessonterm}`, `{langtype}`,
  `{classDatetime}` — 레거시 `pd_reg_*_2` 템플릿에서 이미 사용 중
- `{rewardDays}` — 스냅된 기간 연장 일수, **두 플랜 모두**에 설정됨
  (무제한 및 라이트 루틴)
- `{rewardCount}` — 스냅된 보너스 레슨 횟수, **오직 라이트 루틴에서만** 설정됨.
  무제한 템플릿에서는 참조되지 않음
- `{deadlineDaysLeft}` — 활성 마감일까지 남은 정수 일수
  (N5에서만 사용됨)
- `{moHomeLink}` / `{pcHomeLink}` — 레거시 템플릿에서 이미 사용 중이며,
  홈으로의 인증 래핑 리다이렉트 (앱이 예약 여부에 따라 상태 A 또는 B 선택)
- `{moPrestudyLink}` / `{pcPrestudyLink}` — **신규**, 첫 레슨 `booking_id`에
  해당하는 예습 화면으로의 인증 래핑 리다이렉트

**알림톡 (라이트 루틴):** 무제한과 동일한 구조이며, 구분선 블록만 아래로 교체:

## 혜택 "완료" 조건

보너스는 유저가 첫 레슨을 **완료**하고, 그 레슨의 `scheduled_end_at`이 활성
혜택 기간 안에 들어올 때에만 발동됩니다. 예약만 한 것으로는 부족하며,
튜터의 정산(finalize)이 늦어지더라도 유저가 불이익을 받지는 않습니다.

```
   booked                 ──────►  아직 보너스 없음
       │
       ↓
   lesson starts          ──────►  아직 보너스 없음
       │
       ↓
   lesson ends (wall)     ──────►  아직 보너스 없음
       │
       ↓
   tutor finalizes in     ──────►  ✅ scheduled_end_at ≤ 활성 마감일
   grape (COMP_DATETIME)            일 때에만 보너스 지급
                                    │
                                    ↓
                                    무제한:     +N일
                                    라이트 루틴: +N일 AND +M회
                                    (N4 푸시 + 알림톡)
```

- **트리거** = 튜터가 `grape`에서 수업을 정산 처리 (`GT_CLASS.CLASS_STATE = FINISH`, `INVOICE_STATUS = COMPLETED`, `COMP_DATETIME` 기록)
- **자격 비교 기준** = 레슨의 `scheduled_end_at`과 저장된 `active_deadline` (`COMP_DATETIME`이 아님).
  이로써 튜터의 정산 지연으로부터 유저를 보호합니다. 예를 들어 마감일 당일
  23:30~23:55로 예약된 레슨은 `COMP_DATETIME`이 다음 날 00:02에 찍히더라도
  여전히 보너스 지급 대상입니다
- 레슨의 `scheduled_end_at`이 활성 기간을 벗어나면 보너스 없음
- 유저가 day 7이 끝날 때까지 단 하나의 레슨도 완료하지 않으면 보너스는 영구히 소멸

현재 시스템 기준 구현 노트:
- 레슨 완료는 현재 `grape`에 기록되며, 수업 정산 시 `GT_CLASS.CLASS_STATE = FINISH`, `GT_CLASS.INVOICE_STATUS = COMPLETED`가 업데이트되고 `GT_CLASS.COMP_DATETIME`이 찍힙니다. 정산 이벤트는 **트리거**이고, 예약의 `scheduled_end_at` 필드는 **비교** 입력값입니다.
- 보너스 상태는 `purchase_id` 단위로 추적되며, 구매 시점에 절대 UTC 마감일 + 유저 타임존을 스냅샷으로 기록합니다. 스냅된 타임존은 자격 판정, 연장 스케줄링, 알림 타이밍, 인앱 `오늘 / 내일` 레이블 모두의 단일 진실 소스(source of truth)입니다.
- 미지급 상태의 `purchase_bonus` 레코드가 동시에 여러 개 활성인 경우(예: 첫 실제 구매 + 어드민 재지급이 얹힌 경우), 완료 이벤트는 `created_at` 기준 **가장 최근** 활성 레코드에 바인딩합니다 — 이는 홈 카드/토스트의 우선순위 규칙과 일치하므로, 유저가 본 카드가 곧 지급받는 카드가 됩니다.
- 지급 대상 완료가 감지된 이후의 리워드 지급은 앱 UI가 아니라 `podo-backend`에서 수행되어야 합니다.
- 무제한 리워드: 기존 백엔드 만료일/종료일 업데이트 경로를 사용해 유저의 활성 엔타이틀먼트 종료일을 연장합니다.
- 라이트 루틴 리워드: 엔타이틀먼트 종료일을 연장함과 **동시에** 동일 원자 트랜잭션 내에서 BONUS subscribe/ticket 레코드를 생성/업데이트합니다 (기존 백엔드의 보너스 엔타이틀먼트 인프라 사용). 두 작업은 함께 성공해야 하며, 그렇지 않으면 지급 전체가 롤백되어야 합니다.
- 프라이머리 트리거는 지연된 전체 스캔이 아니라 **완료된 레슨 각각에 대한 이벤트 기반**입니다. 각 완료 이벤트는 지급 체크를 안전하게 시도할 수 있는데, purchase-bonus 레코드가 멱등(idempotent)이기 때문입니다.
- 다만 초기 기간의 자동 연장, 리마인더 발송, 그리고 레슨은 완료 처리되었지만 지급 호출이 실패한 경우를 보정하기 위한 저빈도 reconciliation 작업을 위해서는 크론이 여전히 필요합니다.
- 이 기능은 **서버 제어 킬 스위치** 뒤에 있어야 합니다. 앱 UI와 백엔드 지급 로직 모두에 게이트가 걸려 있어야 퍼널을 깨끗하게 꺼 둘 수 있습니다.
- 보너스 수량은 `grape` 어드민에서 편집 가능해야 하며, 구매 시점에 purchase-bonus 레코드로 **스냅샷**되어야 어드민이 나중에 값을 수정하더라도 이미 약속된 리워드가 조용히 바뀌지 않습니다.
- N4는 백엔드 지급이 성공한 이후에만 발송되어야 하며, 지급 플로우 전체가 구매 단위로 멱등이어야 합니다.
- `PodoScheduleServiceImplV2.book()` 내부에서, 유저가 첫 번째 정규 레슨을 예약할 때에는 먼저 활성 미지급 상태의 `purchase_bonus`가 있는지 확인해야 합니다. 있다면 → 새 N1(혜택 기간 내) 또는 N2(혜택 기간 외) 템플릿으로 라우팅합니다. 없다면 → 기존 `pd_reg_weeklyclass_2` / `pd_reg_infinity_2` 템플릿을 변경 없이 그대로 사용합니다.

라이트 루틴 카피 노트:
- 혜택 카드 예시는 `21일 연장 + 5회 추가 혜택` (3개월), `30일 연장 + 8회 추가 혜택` (6개월), `60일 연장 + 12회 추가 혜택` (12개월)으로 바뀝니다
- 홈 토스트 예시는 `이용 기간 21일 연장 + 보너스 레슨 5회 드려요!`로 바뀝니다
