# 디퍼드 NPS 프롬프트 (Deferred NPS Prompt)

*Created At: 2026-04-28T10:17:02.171316+00:00*

## 목표 (Goal)

수업 종료 후 사용자가 다음에 앱을 열어 /home에 도달했을 때 NPS 설문을 노출하여, 실제 PODO 수업 대비 NPS 응답 도달률을 약 7%에서 50% 이상으로 끌어올린다 (7배 향상). 현재 인-플로우의 '나가기' 버튼을 누르지 않아 누락되는 약 93%의 수업을 회수하는 것이 핵심이다.

## 사용자 스토리 (User Stories)

1. **수업 도중 앱을 끄거나 OS 뒤로가기로 나간 학생**으로서, 다음에 앱을 열고 /home에 도달했을 때 방금 끝난 수업의 NPS 프롬프트를 보고 싶다. 그래야 정확한 인-플로우 종료 경로를 따르지 않아도 수업에 대한 피드백을 남길 수 있다.
2. **앱을 다시 열기 전에 여러 수업을 들은 학생**으로서, 가장 최근에 끝난 적격 수업에 대해서만 NPS 프롬프트를 보고 싶다. 그래야 오래된 수업까지 줄줄이 설문이 뜨는 부담이 없다.
3. **결제 또는 차단 화면에 있는 학생**으로서, /home으로 이동하기 전까지는 NPS 프롬프트로 인해 흐름이 끊기지 않기를 원한다. 그래야 결제·예약·온보딩 같은 핵심 흐름이 절대 방해받지 않는다.
4. **수업을 평가하고 싶지 않은 학생**으로서, '평가하지 않고 나가기' 텍스트 링크를 명시적으로 눌러 해당 수업의 프롬프트를 영구히 닫고 싶다. 그래야 평가하지 않기로 한 수업에 대해 반복적으로 물어보지 않고, 시스템이 내 선택을 존중하게 된다.
5. **PM / 분석팀**으로서, 14일 단위로 (제출 + 명시적 스킵) / 전체 실 수업 비율로 NPS 도달률을 측정하고 싶다. 그래야 디퍼드 프롬프트가 실제로 피드백 격차를 좁히는지, 자기선택 편향(self-selection bias)이 있는지 추적할 수 있다.

## 제약 (Constraints)

- 프롬프트는 오직 /home 페이지에서만 노출된다 — 훅(hook)은 /home 페이지 또는 /home만 감싸는 레이아웃에 배타적으로 마운트되며, 글로벌 앱 셸에는 마운트하지 않는다.
- 차단 화면: 클래스룸 URL, 로그인, 온보딩, 결제, 체크아웃, /my-podo, /reservation, 그 외 /home이 아닌 모든 경로에서는 절대 노출하지 않는다.
- /home에서 모달/오버레이가 열려 있는 동안에는 절대 노출하지 않는다 — 기존 글로벌 오버레이 상태(overlay.isOpen())를 사용한다. 모달이 열려 있으면 프롬프트를 큐잉하고 overlay.isClosed를 감시한 뒤 노출한다.
- 사용자가 10분 내에 시작 예정인 수업이 있으면 노출하지 않는다.
- 세션 캡: 앱 세션당 디퍼드 프롬프트는 최대 1회. 단, 명시적 사용자 액션(점수 제출 OR 스킵 버튼 클릭)에 의해서만 캡이 소진된다. 조용한 닫힘(다른 화면으로 이동)은 캡을 소진하지 않는다.
- 수업 캡: 한 수업당 디퍼드 프롬프트 최대 1회 (nps_skip 또는 nps_response 행으로 강제됨).
- 우선순위: 가장 최근에 끝난 적격 수업 1건만 노출. 더 오래된 적격 수업은 조용히 버려진다 (24시간 윈도우에서 만료).
- 적격 윈도우: 수업의 예정 종료 시각이 과거이면서 동시에 지난 24시간 이내여야 한다.
- 적격 조건: 실제 PODO 수업(prestudy/AI 아님), 학생이 해당 수업의 소유자, 취소/노쇼 아님, 출석 신호 존재, 이전 NPS 응답 행 없음, 이전 NPS 스킵 행 없음.
- 출석 신호는 OR 의미론으로 동작: 해당 수업의 `meet_connected` 이벤트(또는 `meet_participant_joined` 카운트 ≥ 2) OR CLASS_STATE='FINISH'. 둘 중 하나라도 참이면 적격. 둘 다 없으면 적격이 아니다 (수업에 참여하지 않은 사람에게 묻는 것보다 누락이 낫다).
- `meet_connected`는 출시 시점에 필수이며, v2로 미루지 않는다 — 튜터 수동 완료 지연이 지배적인 케이스이기 때문이다 (중앙값 1분, 평균 약 9분, 롱테일은 수 시간). `meet_connected`가 없으면 튜터가 완료를 누르기 전에 사용자가 앱을 포그라운드한 모든 케이스를 조용히 누락시키게 되며, 이는 타깃 코호트의 대부분이다. 클라이언트 출시 전에 출석 미러 테이블(또는 동등한 저비용 쿼리 경로)을 반드시 구현해야 한다.
- CLASS_STATE='FINISH'는 부차적인 OR 분기로, `meet_connected` 이벤트는 유실되었지만 튜터가 완료를 표시한 드문 케이스를 보완한다 (벨트와 멜빵).
- API 호출 전략: getPendingNps 결과를 세션당 1회 캐싱. 어텐션 회복(네이티브 background→active postMessage OR 웹 Page Visibility API의 visibilitychange hidden→visible)이 발생하면 캐시를 무효화. 무효화 후 다음 /home 마운트 시 재호출. /home이 가시 상태로 유지되는 동안의 단순 인앱 탭 전환에는 API 호출하지 않음.
- 프롬프트가 떠 있는 상태에서의 화면 이탈(nav-away): 조용히 닫힘 — 스킵 행을 쓰지 않으며, 수업은 적격 상태를 유지하고 동일 세션 내에서 /home으로 돌아오면 다시 노출된다.
- 조용한 닫힘 후 재노출: 명시적 액션으로 세션 캡이 소진될 때까지 동일 세션 내 모든 /home 복귀에서 프롬프트가 다시 뜬다.
- 콜드 스타트(앱 강제 종료 후 새 실행)는 background→foreground와 동일한 흐름을 트리거한다 — 사용자가 어떤 경로로 들어왔든 훅은 /home에서 마운트된다.
- 출석 신호 하한 버퍼 없음 — now >= class_end가 되는 즉시 트리거하며, 신호는 초 단위 내에 도착한다고 가정한다.
- getPendingNps 레이턴시 예산: p95 100ms.
- getPendingNps 및 /nps/skip 에러율: 1% 미만.
- 기존 /review-complete 페이지를 ?source=deferred 쿼리 파라미터와 함께 재사용한다.
- NPS 타이틀 카피를 두 경로(인-플로우 및 디퍼드)에서 모두 '오늘 레슨, 어떠셨나요?' → '이번 레슨, 어떠셨나요?'로 변경한다 — 단일 정본(canonical) 카피. 서브타이틀 '솔직한 평가는 더 좋은 레슨을 만드는 데 큰 힘이 돼요.'는 변경하지 않는다.
- 두 경로 모두에서 서브타이틀 아래에 새로운 튜터 컨텍스트 카드를 렌더링한다. 레이아웃: 좌측 아바타 원 + 우측 스택 텍스트(상단 튜터 이름, 하단 수업 날짜/시간)로 구성된 알약/카드. 예: 'A' 아바타(주황색 배경, 흰색 글자) + 'Alice' + '4월 26일 11:00'.
- 아바타 폴백 규칙: 실제 프로필 사진이 도입되기 전까지 아바타는 튜터 표시 이름의 첫 글자를 흰색 글자로 표기한 단색 주황색 원으로 한다. 영문 이름(Alice → 'A')과 한글 이름(김철수 → '김') 모두에 동작한다. 이후 실제 프로필 사진이 마련되면 첫 글자 대신 사진이 들어가며, 본 규칙은 v1 폴백이다.
- 카드에 표시되는 날짜/시간은 KST 기준 수업 시작 시각이며, 포맷은 '{M}월 {D}일 {HH}:{MM}'.
- 튜터 이름, 튜터 프로필(존재 시), 수업 시작 일시는 /review-complete 페이지에서 사용 가능해야 한다. 기존 데이터 페치에 모두 포함되지 않을 수 있으므로 필요 시 검증 및 API 확장(신규 getPendingNps 응답에는 이미 tutor_name과 class_end_datetime_unix가 포함되어 있고, class_start_datetime은 추가가 필요).
- 스킵 버튼: 페이지 하단의 3차(tertiary) 텍스트 링크로 렌더링한다 (기본 CTA 아래의 회색 버튼이 아님). 카피: '평가하지 않고 나가기'. 클릭 시 (a) POST /api/v1/lesson-review/nps/skip를 통해 nps_skip 행을 쓰고, (b) 프롬프트를 닫고, (c) 세션 캡을 소진하며, (d) 크로스-탭 락을 해제한다. 해당 수업은 적격 대상에서 영구히 제외된다.
- 크로스-탭 조정: 동일 classId에 대해 모든 열린 탭을 통틀어 NPS 표면(디퍼드 프롬프트 OR /review-complete)은 한 번에 하나만 활성화될 수 있다. 구현은 BroadcastChannel(우선) + localStorage 폴백. 어떤 탭이든 NPS 표면을 열 때 `nps_prompt_active:{classId}={timestamp}`를 60초 TTL과 함께 브로드캐스트/기록 (탭 크래시 대비). 다른 탭은 자기 표면을 노출하기 전에 이 락을 읽고, 락이 있으면 노출을 억제한다. 제출/스킵/명시적 닫힘 시 락을 해제한다.
- /review-complete 인-플로우 경로도 크로스-탭 락을 존중한다: 사용자가 클래스룸에서 '나가기'를 누르면, 기존 isClassEnded 게이트도 크로스-탭 락을 함께 검사한다. 다른 탭이 해당 수업을 평가 중이면 /review-complete가 아니라 /home으로 직접 라우팅한다 — 조용한 우회이며, 중간의 "다른 탭에서 평가 중" 같은 페이지를 노출하지 않는다.
- 다중 탭 동시 노출 방지: /home 탭에 디퍼드 프롬프트가 떠 있는 상태에서 사용자가 다른 탭에서 /review-complete를 열면, /review-complete가 락을 보고 스스로 /home으로 라우팅한다. /review-complete가 먼저 마운트되어 있고 사용자가 이후 /home으로 포커스를 옮기면, /home이 락을 보고 디퍼드 프롬프트를 억제한다.
- 롤아웃: 백엔드 선배포 → 클라이언트는 피처 플래그 tbd_260X_nps_deferred_prompt 뒤에 배포 → 출시 시 100%로 켠다(점진 램프 없음). 킬 스위치는 플래그 OFF.

## 성공 기준 (Success Criteria)

1. 1차: 14일 윈도우 기준 NPS 도달률(제출 + 명시적 스킵)/전체 실 PODO 수업 ≥ 50%, 출시 후 30일 이내 측정 (현재 제출 베이스라인 약 5.3% 대비).
2. 2차: 디퍼드 경로의 스킵율이 인-플로우 경로의 스킵율 대비 유의미하게 높지 않을 것.
3. 2차: 디퍼드 경로의 평균 NPS 점수가 인-플로우 평균과 유의미하게 다르지 않을 것.
4. 2차: /home p95 레이턴시가 getPendingNps 100ms 예산을 넘는 회귀가 없을 것.
5. 2차: getPendingNps 및 /nps/skip 에러율 1% 미만.
6. 2차: 제출 / 전체 실 수업 (현재 베이스라인과의 사과 대 사과 비교 — 명시적 스킵 트래킹은 신규).

## 가정 (Assumptions)

- 대부분의 사용자는 수업 후 24시간 이내에 앱을 포그라운드하므로, '가장 최근 1건만, 나머지는 폐기' 정책이 50% 도달률 목표를 의미 있게 훼손하지 않는다.
- 대부분의 세션이 /home을 거치므로, 훅을 /home에만 한정해도(더 넓은 안전 화면 리스트가 아니라) 충분한 커버리지가 나온다.
- 출석 신호(meet_connected 또는 CLASS_STATE='FINISH')는 수업 종료 후 초 단위 내에 도착한다 — 분 단위의 지연은 예상하지 않는다.
- 기존 글로벌 오버레이/모달 상태(overlay.isOpen() / overlay.isClosed)는 프롬프트 노출 게이트로 쓰기에 충분히 신뢰성·포괄성이 있다.
- 기존 /review-complete 페이지는 ?source=deferred 쿼리 파라미터로 디퍼드 경로를 구분하며, 그대로 재사용 가능하다.
- 도달률 분모(학생+튜터가 모두 미트에 입장하고 취소·노쇼가 아닌 모든 실 PODO 수업)는 안정적으로 쿼리 가능한 데이터셋이다.
- 현재 NPS 도달 베이스라인은 약 7% (14일 동안 약 1,248 / 16,778 실 수업).
- 타깃 브라우저에서 BroadcastChannel API를 사용할 수 있으며, 부족분은 localStorage 폴백이 보완한다.
- 다중 탭 경합은 웹에서만 발생한다 — 네이티브 WebView는 단일 표면이므로 락은 웹 전용이지만, 네이티브에서 읽혀도 무해하다.

## 추후 결정 (Decide Later)

이 단계에서 미루었거나 시기상조로 식별된 항목들. 더 많은 컨텍스트가 확보되면 다시 들여다본다.

- 출시 후 분석에서 출석 신호 지연으로 인한 누락이 무시할 수 없는 수준으로 나오면, 클라이언트 사이드 소프트 재시도 또는 그레이스 버퍼 추가를 재검토한다.
- 정확한 피처 플래그 이름 (플레이스홀더: tbd_260X_nps_deferred_prompt).
- 실제 튜터 프로필 사진이 도입되었을 때의 아바타 처리 방식 (현 v1: 첫 글자 주황색 원).
- 세션당 두 건 이상의 수업 NPS 노출 (연속 설문은 단순성을 위해 거부).
- 더 최근 수업이 우선순위를 가져갈 때 폐기되는 더 오래된 적격 수업의 회수.
- 점진적 롤아웃 램프 — 플래그 뒤에서 100%로 출시, 퍼센트 램프 없음.
- /home 외의 화면(예: /my-podo, /reservation)에서의 디퍼드 프롬프트 노출.
